# PDF Pro Tool 업데이트 배포 — Claude Code 프롬프트 템플릿

## 사용법
1. 코드 수정 완료 후
2. 아래 프롬프트에서 ✏️ 부분만 수정
3. Claude Code에 붙여넣기
4. 끝나면 GitHub에서 Release만 수동 생성

---

## 프롬프트 (아래를 통째로 복사해서 Claude Code에 붙여넣기)

```
PDF Pro Tool 업데이트 배포를 진행해줘. 아래 순서대로 실행해줘.

✏️ 이번 업데이트 정보:
- 새 버전: 1.1.0
- 변경 내용: 여기에 수정한 내용 간단히 작성

순서:

1. version.py의 __version__을 위에 적은 새 버전으로 수정해줘.

2. Git에 커밋하고 push해줘.
   - 커밋 메시지: "v{새 버전}: {변경 내용}"

3. PyInstaller로 빌드해줘.
   - PDFProTool.spec 파일 사용
   - 빌드 완료 후 dist 폴더의 exe 파일 경로와 파일 크기를 알려줘.

4. 빌드 완료되면 GitHub Release 생성 명령어를 알려줘:
   - tag: v{새 버전}
   - title: v{새 버전}
   - 변경사항 내용 포함
   - exe 파일 첨부

각 단계 완료할 때마다 결과를 알려줘.
```

## Claude Code 작업 완료 후 내가 할 일 (마지막 수동 작업)

### GitHub Release 생성 (2가지 방법 중 택1)

**방법 A: GitHub 웹사이트에서 수동 생성**
1. https://github.com/jang375/PDFProTool/releases/new 접속
2. Tag: v1.1.0 입력
3. Title: v1.1.0
4. 설명에 변경사항 작성
5. dist 폴더의 exe 파일 드래그해서 첨부
6. "Publish release" 클릭

**방법 B: PowerShell에서 gh 명령어 (GitHub CLI 설치 필요)**
```powershell
gh release create v1.1.0 "dist\PDFProTool.exe" --title "v1.1.0" --notes "변경사항 내용"
```
